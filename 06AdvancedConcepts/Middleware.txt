What is Middleware in FastAPI?

Middleware is a function that wraps every request and every response.

It runs:

Before the request reaches your route handler.
After the route handler returns a response.

This is an important difference: in FastAPI, middleware can execute code both before and after the route handler.




 Request
   │
   ▼
Middleware
   │
   ▼
await call_next(request)
   │
   ▼
Route Handler
   │
   ▼
Response
   │
   ▼
Back to Middleware



| Middleware                                              | Route Handler                                                  |
| ------------------------------------------------------- | -------------------------------------------------------------- |
| Runs before every request reaches the route handler     | Runs only when its specific endpoint is requested              |
| Can also run after the response is created              | Creates the response for that endpoint                         |
| Used for logging, authentication, timing, CORS, etc.    | Contains the business logic (fetch users, create orders, etc.) |
| Calls `await call_next(request)` to continue processing | Returns the response (JSON, HTML, file, etc.)                  |
